Create a TypeScript file called machine.ts that defines a generic state machine class Machine and an example media player state machine, mediaPlayerMachine, with parallel and nested states. Here's the detailed specification:

Generic State Machine Class:

Define a class Machine that accepts three generic types:
TState: A string union representing state names.
TEvent: A type for events, which is an object with a type property and an optional data property.
TContext: A type for context, representing the state machine's data.
The class should have methods:
send(event: TEvent): Triggers a transition based on the current state and the event.
getState(): Returns the current state(s).
getContext(): Returns the current context.
onTransition(callback: (state: TState[]) => void): Allows subscribing to state transitions.
offTransition(callback: (state: TState[]) => void): Allows unsubscribing from state transitions.
State Definition:

The class should use a StateDefinition type, which includes:
transitions: A record mapping event types to transitions.
enter: A function that runs when entering the state, receiving the current context and a send function.
after: A record for delayed transitions.
states: An object for nested states.
parallel: A boolean indicating if the state has parallel sub-states.
State Transition:

A transition includes a target state and an optional action that updates the context.
Media Player Example:

Create a mediaPlayerMachine with the following state hierarchy:
Top-level parallel states: playback and network.
playback has three nested states: stopped, playing, paused.
network has two nested states: connected and disconnected.
The machine should handle the following events:
PLAY event with data (e.g., trackId, trackName).
PAUSE, STOP, CONNECT (with connection type), and DISCONNECT.
Example Usage:

Demonstrate how to trigger transitions with events such as PLAY, PAUSE, and CONNECT.
Log the transitions and current state.
The final code should follow this structure and work in TypeScript without type errors.
